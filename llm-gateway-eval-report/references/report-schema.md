# Report Artifact Schema

Use this reference when maintaining `scripts/render_gateway_eval_report.py`.

## Codex Compare Run

Expected directory:

```text
runs/<slug>-codex/
├── summary.json
└── results.jsonl
```

Important `summary.json` fields:

- `total_runs`
- `overall_pass_rate`
- `targets_summary.official`
- `targets_summary.gateway`
- `paired_task_deltas`
- `pricing_comparison`
- `pricing_note`

Each target summary may contain:

- `runs`
- `passed`
- `pass_rate`
- `codex_failures`
- `verify_failures`
- `avg_codex_duration_seconds`
- `usage.input_tokens`
- `usage.cached_input_tokens`
- `usage.output_tokens`
- `usage.reasoning_output_tokens`
- `usage.total_tokens`
- `estimated_cost`
- `currency`
- `cost_per_pass`

Each `results.jsonl` row may contain:

- `task_id`
- `target`
- `repeat_index`
- `passed`
- `codex_exit_code`
- `verify_exit_code`
- `codex_duration_seconds`
- `verify_duration_seconds`
- `usage`
- `error`
- `workspace`
- `patch_path`

## Gateway Quick Eval Run

Expected directory:

```text
runs/<slug>-security/
├── audit.json
├── summary.json
├── results.jsonl
└── security_probes.json
```

Important `summary.json` fields:

- `providers.gateway.total`
- `providers.gateway.request_success_count`
- `providers.gateway.request_success_rate`
- `providers.gateway.case_pass_count`
- `providers.gateway.case_pass_rate`
- `providers.gateway.error_count`
- `providers.gateway.error_types`
- `providers.gateway.network_error_count`
- `providers.gateway.model_invocation_error_count`
- `providers.gateway.model_name_mismatch_count`
- `providers.gateway.assertion_pass_rate`
- `providers.gateway.json_schema_pass_rate`
- `providers.gateway.usage_present_rate`
- `providers.gateway.latency.p50`
- `providers.gateway.latency.p95`
- `providers.gateway.latency.p99`
- `providers.gateway.tokens_per_second`
- `providers.gateway.security_case_pass_rate`
- `providers.gateway.model_authenticity_case_pass_rate`
- `providers.gateway.overall_score`

Important `security_analysis` fields:

- `generated_probe_pass_rate`
- `canary_leakage_event_count`
- `tamper_probe_failure_count`
- `usage_anomaly_count`
- `model_mismatch_count`
- `honeytoken_count`
- `external_honeytoken_event_count`
- `external_honeytoken_status`

Important `audit.json` fields:

- `summary.PASS`
- `summary.WARN`
- `summary.FAIL`
- `findings[]`

Each finding contains:

- `target`
- `check`
- `status`
- `message`
- `evidence`

## Optional Benchmark Run

The HTML renderer accepts `--benchmark-run` pointing to either:

- a benchmark plan JSON generated by `llm-gateway-eval benchmark-plan`
- a normalized result summary JSON generated by `llm-gateway-eval benchmark-import`
- a directory containing `summary.json`

Plan payload:

- `kind: benchmark_plan`
- `status: planned`
- `difficulty`
- `suites[]`

Each plan suite contains:

- `benchmark`
- `name`
- `category`
- `official_url`
- `best_for`
- `recommended_sample_size_per_target`
- `runner_hint`
- `result_import.accepted_formats`

Result payload:

- `kind: benchmark_results`
- `status: ready`
- `sources[]`
- `targets_summary[]`
- `benchmark_summary[]`
- `results[]`

Each normalized result row contains:

- `benchmark`
- `suite`
- `task_id`
- `target`
- `passed`
- `score`
- `duration_seconds`
- `difficulty`
- `model`
- `error`

Each summary row contains:

- `tasks`
- `scored_tasks`
- `passed`
- `failed`
- `unknown`
- `pass_rate`
- `avg_score`
- `avg_duration_seconds`
- `error_types`
